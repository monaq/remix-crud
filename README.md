# Remix QnA Board

Remix Real-world Example

## 개발 환경 설정

```bash
pnpm install
```

## 개발 서버 실행

```bash
pnpm run dev
```

## 커밋 컨벤션

이 프로젝트는 [Conventional Commits](https://www.conventionalcommits.org/)를 따릅니다.

### 커밋 타입

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅, 세미콜론 누락 등 (기능 변경 없음)
- `refactor`: 코드 리팩토링
- `test`: 테스트 추가 또는 수정
- `chore`: 빌드 프로세스 또는 보조 도구 변경

### 커밋 예시

```bash
pnpm run commit
```

또는

```bash
git commit -m "feat: QnA 게시글 작성 기능 추가"
git commit -m "fix: 로그인 상태 확인 로직 수정"
git commit -m "docs: README 업데이트"
```

## 스크립트

- `pnpm run dev`: 개발 서버 실행
- `pnpm run build`: 프로덕션 빌드
- `pnpm run start`: 프로덕션 서버 실행
- `pnpm run typecheck`: TypeScript 타입 체크
- `pnpm run commit`: 대화형 커밋 (Commitizen 사용)
